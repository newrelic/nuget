New Relic Nuget Packages

nuget-azure-cloud-services

The package that includes the latest New Relic x64 installer, so that you can easily include New Relic in your Windows Azure deployment.

github: 

> git clone git@github.com:newrelic/nuget-azure-cloud-services.git

nuget:

PM> Install-Package NewRelicWindowsAzure

nuget-azure-web-sites

The nuget package for x86 apps (of type "web site") running on Windows Azure.

github:

> git clone git@github.com:newrelic/nuget-azure-web-sites.git

nuget:

PM> Install-Package NewRelic.Azure.WebSites

nuget-azure-web-sites-x64

The nuget package for x64 apps (of type "web site") running on Windows Azure.

github:

> git clone git@github.com:newrelic/nuget-azure-web-sites-x64.git

nuget:

PM> Install-Package NewRelic.Azure.WebSites.x64

nuget-agent-api

The New Relic .NET Agent API supports custom error and metric reporting, and custom transaction parameters. If the agent is not installed or is disabled, method invocations of this API will have no effect. You can find more information about the NewRelic.Agent.Api at http://newrelic.com/docs/dotnet/AgentApi.

github:

> git clone git@github.com:newrelic/nuget-agent-api.git

nuget:

PM> Install-Package NewRelic.Agent.Api

Contribute

Setting up your local environment to contribute

  1. Open visual studio
  2. Tools menu >> Library Package Manager >> Package manager settings
  3. Under the Package Manager, click on Package sources
  4. Create a source named "Local" and set the directory to where you cloned the repositories (something like "c:\code\newrelic_nuget_packages")
  5. Click Add then "OK"

Using the packages locally

  1. To use it you can open up your web app and choose >> tools >> Library Package Manager >> Package manager console
  2. In the console make sure that the source drop down is set to local
  3. Then type in "Install-Package [PACKAGE-NAME]"
  4. Hit enter
